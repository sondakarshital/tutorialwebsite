Async parallel ,series ,waterfall

As an nodejs or javascript developer it's always essential to know about the async capability provided by async module provided by node. In this post we will be going through async parrallel,series,waterfall models.

As i started using async feature of node in the begining failed to understand the exact purpose of this module. Using thease modules frequntly made me understand them better. Let's start with it !!!.

Need to run multiple tasks that depend on each other and when they all finish do something else.
In this scenario we have to use async waterfall. Well now how to use this ?.

async.waterfall()
We have to require async module in order to use it in our code .Let's do that first.
var async = require('async');
 The syntax for waterfall looks like async.waterfall(tasks, callback), where tasks is an array of functions. It calls the array of function present in the tasks array one after the other by passing output through callback of first function to the next in the tasks array and so on. Once the final function in array returns its response without any error , async calls the final function with success response. If error at any point of time while executing functions present in task array immediatly final function with error will be called by skipping successive function calls.

The main diffrence between async series and waterfall is, async series is no where dependent on privious function output but the waterfall does.

When to use which?

Async Series - when we have a requirenment where we need to call all the funtions in series where each funtion call is independent of each other.

Async Parallel - we have a requirnment where we need to call all the functions in task array at a time where each function call is independent of other.

Async Waterfall - we have a requirenment where we need to call the funtions one after the other passing output of one funtion as input to the successive funtion in task array.
